# Отчет по лабораторной работе №2

## Цель данной лабораторной работы — освоить практические навыки автоматизации процесса непрерывной интеграции и развертывания (CI/CD) с использованием Jenkins — мощного инструмента автоматизации сборки, а также наладить взаимодействие с удалённым репозиторием GitHub посредством webhook, обеспечивающим мгновенный запуск сборки при каждом изменении кода.

## Ход работы:

1. Авторизовались на Jenkins-сервере по указанному IP и порту.

<img width="258" height="113" alt="image" src="https://github.com/user-attachments/assets/f184aed2-c787-4d54-9aba-acfbe19a7366" />

2.  Создаем  новый проект и задаем уникальное имя проекта. Далее в списке типов проектов выбераем Pipeline и нажимаем OK.

3. В открывшемся окне переходим к разделу Pipeline:
  3.1.	В поле Definition выбераем Pipeline script. В поле ниже вставляем заготовленный Pipeline скрипт (Jenkinsfile), предоставленный преподавателем. 
  3.2.	В Jenkinsfile отдельно редактируем параметры: В поле STUDENT_NAME указываем своё имя, в поле PORT укажите уникальный порт.

   <img width="1329" height="573" alt="image" src="https://github.com/user-attachments/assets/4e676612-868a-40a1-bd7d-8e241f18b379" />

5. Далее настраиваем GitHub webhook для автоматического запуска сборки.

   <img width="786" height="798" alt="image" src="https://github.com/user-attachments/assets/1fd985d8-070e-4036-83be-ef3a12ce3c8b" />

7. В интерфейсе Jenkins открываем созданную job. Нажмаем кнопку Build Now, чтобы инициировать первую сборку. Мониторим процесс исполнения через вкладку Console Output — убеждаемся, что код корректно выкачан, прошли тесты, приложение собрано и запущено. Далее открываем приложение в браузере по адресу – http://84.201.147.67:8081/

<img width="1280" height="636" alt="image" src="https://github.com/user-attachments/assets/1ecb8f46-5280-45cd-9a6d-377559af2e05" />

8. Внесли изменения в репозиторий (поменяли цвет и текст) и выполнили коммит в ветку master.

 
9. Убеждаемяь, что GitHub webhook корректно сработал, и Jenkins автоматически запустил повторную сборку.
 
Вывод: Освоили практические навыки автоматизации процесса непрерывной интеграции и развертывания (CI/CD) с использованием Jenkins — мощного инструмента автоматизации сборки, а также наладили взаимодействие с удалённым репозиторием GitHub посредством webhook, обеспечивающим мгновенный запуск сборки при каждом изменении кода.
